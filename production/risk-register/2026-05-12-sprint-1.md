## R-S1-01: Claude API rate limits or token cost during prototype iteration

| Field | Value |
|---|---|
| **Severity** | MEDIUM |
| **Likelihood** | MEDIUM |
| **Status** | OPEN |
| **Opened** | 2026-05-12 |
| **Owner** | user (developer) |
| **Linked to** | Sprint 1 S1.2 |

### Description
Iteration on the prototype will fire repeated requests against the Claude API. Rate-limits or token-cost concerns could throttle iteration speed or rack up bills.

### Impact
Sprint 1 slips; or user de-scopes to mock-only validation (which leaves Q4 + Q5 still unanswered).

### Mitigation
- Use the cheapest available endpoint for iteration (smallest completion or a free-tier route if available)
- Soft budget: ~$20 for the whole sprint
- Switch to mock mode (ADR-0008) for iteration on the *bridge* shape; only hit real API for Q-answering passes

### Trigger
- Day 2: if iteration is gated by rate-limits → switch to mock for most iteration cycles
- Day 3: if $10 already spent → reduce iteration cadence, single-request validation only

## R-S1-02: Real-API payloads don't fit cleanly into 4-state vocabulary

| Field | Value |
|---|---|
| **Severity** | HIGH |
| **Likelihood** | HIGH |
| **Status** | OPEN |
| **Opened** | 2026-05-12 |
| **Owner** | user + technical-director |
| **Linked to** | ADR-0007 BLOCKED, Sprint 1 S1.3 + S1.5 |

### Description
The art-bible-derived `idle / working / completed / errored` vocabulary may not map cleanly to what real APIs report. The prototype may reveal a 5-state, 7-state, or "state-is-a-vector" reality.

### Impact
ADR-0007 may need a richer vocabulary; downstream GDDs (ASM, ACC, AAL, HUD) may need amendments; visual design may need additional states.

### Mitigation
- This is **the prototype's primary discovery deliverable** — not a failure mode but a feature
- Document whatever real-API states surface; map them to the 4-state visual vocabulary even if the source has more
- If the 4-state visual model is wrong, document the gap clearly so ADR-0007 can resolve it

### Trigger
- Day 3 checkpoint: if states found > 6, escalate to systems-designer for vocabulary triage
- If states require continuous values (not discrete), surface in ADR-0007 with explicit "this is a state plus a metric" pattern

## R-S1-03: VERIFY-7 / VERIFY-8 surface engine-API differences from documented behaviour

| Field | Value |
|---|---|
| **Severity** | MEDIUM |
| **Likelihood** | LOW |
| **Status** | OPEN |
| **Opened** | 2026-05-12 |
| **Owner** | technical-director |
| **Linked to** | VERIFY-7, VERIFY-8, ADR-0001 |

### Description
The Data Bridge ADR-0001 asserts behaviour of `HTTPRequest.request_completed` signal signature and `HTTPRequest.timeout` cancellation. If 4.6.2 has changed these vs documented, ADR-0001's transport pattern needs amendment.

### Impact
ADR-0001 may need a small Status: Superseded → new ADR; transport pattern rework limited to a single layer.

### Mitigation
- VERIFY tests in the prototype itself (Sprint 1 S1.4)
- If anomaly found, escalate to godot-specialist for documented mitigation
- Document any 4.6.2 quirk in `docs/engine-reference/godot/breaking-changes.md`

### Trigger
- Day 1: if `request_completed` signal does not match 4-arg signature, escalate immediately
